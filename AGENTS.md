# AGENTS.md

Инструкции для AI-агентов, работающих с репозиторием `FinanceTracker`.

## Цели изменений
1. Сначала корректность и работоспособность.
2. Затем простота и читаемость.
3. Затем минимальный объём изменений.

## Границы
- Не делать «попутный» рефакторинг вне цели задачи.
- Не менять структуру репозитория без явного запроса.
- Новые зависимости добавлять только при явной пользе.

## Что обновлять при изменениях
- Если меняется поведение, конфигурация или запуск — обновить `README.md` и/или `docs/CONFIG.md`.
- При изменениях процесса разработки — обновить `CONTRIBUTING.md`.

## Базовая проверка перед коммитом
Выполнить из корня репозитория:

```bash
python -m compileall src
docker compose config
```

Если команда недоступна в окружении, явно указать это в отчёте.

## Формат PR
Каждый PR должен содержать разделы:
- What changed
- How to verify
- Risks/Open questions
- How to apply

Для `How to apply` приводить команды Docker (обновление, запуск, логи).
